# Team-Coord Integration Guide for dev-flow

## 并行工作流策略

dev-flow 的各阶段可以并行执行，由 team-coord 的 lead 协调。

## Worker 分配

### 分配方案 A: 按阶段分组

```
Lead:
  "我需要你完成这个开发任务。3 个 worker 分别负责：
   Worker A → onboard + audit（项目扫描 + 安全检查）
   Worker B → review（代码审查 6 视角）
   Worker C → changelog + memory（发布准备 + 记忆保存）"
```

### 分配方案 B: 按审查视角

适用于深度代码审查：

```
Lead:
  "我需要你审查这个 PR。每个 worker 负责一个视角：
   Worker A → Security + Performance
   Worker B → Architecture + Code Quality
   Worker C → Test Quality + API Design"
```

## 合并策略

Lead 收到所有 worker 的结果后：

1. **合并审查发现** — 按严重程度排序，去重
2. **合并安全报告** — 汇总漏洞和修复建议
3. **生成综合报告** — 健康评分 + 行动计划

## 输出模板

```markdown
# Dev Flow Report

## Project Overview
{onboard 结果}

## Security Audit
{audit 结果}

## Code Review
{review 结果 — 健康评分 + 发现列表}

## Changelog
{changelog 结果}

## Decisions
{ADR 列表}

## Next Steps
{优先行动项}

---
*Generated by dev-flow with team-coord*
```

## 注意事项

- Lead 不做实际工作，只做分配和合并
- Worker 之间不需要通信，各自独立执行
- 如果某个 worker 失败，Lead 用自己的 context 补充
- 最终报告由 Lead 格式化，确保一致性
