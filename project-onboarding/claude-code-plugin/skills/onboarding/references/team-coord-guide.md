# Team-Coord Integration Guide for project-onboarding

## 并行扫描策略

project-onboarding 的 7 个扫描阶段可以并行执行，由 team-coord 的 lead 协调。

## Worker 分配

### 分配方案 A: 按阶段分组

```
Lead:
  "我需要你扫描项目并生成入门文档。3 个 worker 分别负责：
   Worker A → 目录结构 + 技术栈检测
   Worker B → 构建系统 + 测试框架
   Worker C → CI/CD + 代码风格"
```

### 分配方案 B: 按语言/模块分组

适用于大型 monorepo：

```
Lead:
  "我需要你扫描这个 monorepo。每个 worker 负责一个子包：
   Worker A → packages/core (TypeScript)
   Worker B → packages/cli (Go)
   Worker C → packages/web (React)"
```

## 合并策略

Lead 收到所有 worker 的结果后：

1. **合并技术栈列表** — 去重，取最高版本
2. **合并目录结构** — 合并为完整树
3. **合并命令列表** — 汇总所有可用命令
4. **合并 CI/CD** — 汇总所有流水线
5. **生成综合报告** — 按输出格式组织

## 输出模板

Lead 合并后使用以下模板：

```markdown
# Project Onboarding: {项目名}

{worker_results_merged}

---
*Generated by project-onboarding with team-coord*
```

## 注意事项

- Lead 不做实际扫描，只做分配和合并
- Worker 之间不需要通信，各自独立扫描
- 如果某个 worker 失败，Lead 用自己的 context 补充
- 最终输出由 Lead 格式化，确保一致性
