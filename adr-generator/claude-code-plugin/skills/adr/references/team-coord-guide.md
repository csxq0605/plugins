# Team-Coord Integration Guide for adr-generator

## 团队决策协作

adr-generator 可以与 team-coord 集成，支持团队协作创建和审查架构决策。

## 工作流

### Worker 创建 ADR

```
Worker A 创建 ADR 草稿:
/adr new --title "Use Event Sourcing" --template madr --status proposed
```

### Lead 审查 ADR

```
Lead 审查所有 proposed 状态的 ADR:
/adr list --status proposed

Lead 批准:
/adr accept 0004

Lead 要求修改:
/adr comment 0004 "请补充替代方案的性能对比数据"
```

### 决策会议

```
# Lead 组织决策讨论
"我们需要讨论 ADR-0004 (Event Sourcing)。
 Worker A，请阐述你的方案。
 Worker B，请阐述你认为应该用 CQRS 的理由。"

# 会议结束后
/adr accept 0004
/adr comment 0004 "团队一致同意，2024-01-15 会议通过"
```

## 输出模板

```markdown
# Architecture Decision Record: {标题}

## 参与者
- {Worker A}: 提出方案
- {Worker B}: 审查
- {Lead}: 批准

## 讨论摘要
{关键讨论点}

## 决策
{最终决策}

## 行动项
- [ ] {Worker A}: {任务}
- [ ] {Worker B}: {任务}

---
*Generated by adr-generator with team-coord*
```

## 注意事项

- ADR 创建后状态为 proposed，需要 Lead 批准才能变为 accepted
- Worker 可以创建 ADR，但不能修改状态
- Lead 负责维护 ADR 索引和状态流转
- 团队决策应该在会议中讨论，ADR 只记录结果
